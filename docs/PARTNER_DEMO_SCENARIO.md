# 학온(HAGON) 파트너 데모 시나리오

## 미팅 목적
파트너사(학원운영관리 프로그램 회사)에게 학온 시스템의 API 연동 방식과 실제 동작을 시연하여, 자사 서비스에 학온을 통합했을 때의 가치를 보여줍니다.

---

## 데모 플로우 (약 15분)

### 1단계: 문제 제기 (2분)
**화면: 랜딩 페이지 (hagon-cloud.vercel.app)**

> "학원 강사 계약에서 가장 흔한 분쟁 유형이 뭔지 아시나요?"

- 리스크 진단 버튼 클릭 → 고정급 선택 → 5가지 질문 진행
- 결과 화면에서 "대부분의 학원이 경고~위험 등급" 강조
- **핵심 메시지**: "이 문제를 귀사 고객(학원)이 겪고 있고, 학온이 해결합니다"

### 2단계: 계약서 자동 생성 시연 (3분)
**화면: 대시보드 → 새 계약서**

- 로그인 → 대시보드 진입
- "+ 새 계약서" 클릭 → 마법사 8단계 진행
- **강조 포인트**:
  - 학원 정보 입력 시 자동 검증
  - 급여 입력 시 최저임금 자동 체크
  - 주휴수당 자동 계산
  - 18가지 법적 검증 항목 실시간 적용

### 3단계: 대시보드 현황 파악 (2분)
**화면: 대시보드**

- 알림 카드 설명: 서명 대기 / 만료 임박 / 작성중 / 전체 체결
- 테이블: 강사명, 계약기간, 상태, 서명 상태 한눈에 파악
- **핵심 메시지**: "원장님이 로그인하면 즉시 리스크 현황을 파악할 수 있습니다"

### 4단계: 전자서명 플로우 시연 (3분)
**화면: 대시보드 → 카톡 전송 → 서명 페이지**

- "카톡 요청 전송" 또는 "링크 복사" 클릭
- 대시보드에서 상태가 "작성중" → "서명대기"로 변경되는 것 시연
- 강사 서명 페이지 접속 → 본인확인 → 계약서 전문 확인 → 서명
- 서명 완료 후 대시보드 새로고침 → "체결완료"로 변경 확인
- **핵심 메시지**: "카카오톡 하나로 비대면 계약 체결이 완료됩니다"

### 5단계: 체결 완료 계약서 확인 (2분)
**화면: 체결완료 탭 → 계약서 보기 → /view 페이지**

- 체결완료 탭에서 "계약서 보기" 클릭
- 워터마크("학온 진본 확인") 포함된 계약서 표시
- PDF 다운로드 시연
- **핵심 메시지**: "법적 효력을 갖춘 전자계약서가 안전하게 보관됩니다"

### 6단계: 파트너 API 연동 설명 (3분)
**화면: 파트너 API 문서 (docs/PARTNER_API_GUIDE.md)**

- 연동 방식 설명:
  1. 파트너사 백엔드에서 학온 API로 세션 생성 (학원 사업자번호로 식별)
  2. 학원 사용자가 SSO로 학온 진입 (별도 로그인 불필요)
  3. 계약서 작성/서명 완료 시 Webhook으로 결과 수신
- SSO 학원 식별 방식:
  - 파트너사가 세션 생성 시 academy_name + biz_number(사업자번호) 전송
  - 사업자번호가 학원별 고유 식별자 역할
  - 프랜차이즈도 지점별 사업자번호가 다르므로 완벽히 분리
  - 파트너사의 인증을 신뢰(trust)하는 구조
- 기술 스펙:
  - HMAC-SHA256 인증
  - Webhook 재시도 (5회, exponential backoff)
  - DB 기반 동적 CORS (파트너사 도메인 자동 관리)
- **핵심 메시지**: "귀사 서비스에 학온을 임베드하면, 고객에게 법률리스크 관리 기능을 즉시 제공할 수 있습니다"

## 예상 질문 & 답변

### Q: 법적 효력이 있나요?
A: 전자서명법에 의거, 당사자 간 합의에 의한 전자서명은 법적 효력을 갖습니다. 학온은 서명 일시, IP 주소, 본인확인 과정을 모두 기록합니다.

### Q: 기존 계약서 양식을 그대로 쓸 수 있나요?
A: 현재는 학온이 1,200건 분쟁 데이터를 기반으로 설계한 표준 양식(TYPE_A)을 제공합니다. 향후 커스텀 템플릿 기능을 추가할 예정입니다.

### Q: API 연동에 얼마나 걸리나요?
A: 기본 연동(세션 생성 + Webhook 수신)은 1~2일이면 충분합니다. 파트너 API 가이드 문서에 JavaScript/Python 예시 코드가 포함되어 있습니다.

### Q: 데이터 보안은 어떻게 되나요?
A: Supabase(AWS 기반) 인프라를 사용하며, Row Level Security(RLS)로 학원별 데이터 격리를 보장합니다. HMAC-SHA256 서명 인증으로 API 보안을 확보합니다.

### Q: 가격 정책은?
A: 파트너사 연동의 경우 학원당 월정액 또는 계약서 건당 과금 방식을 협의 가능합니다. 파일럿 기간에는 무료로 제공합니다.

### Q: 학원 외 다른 업종에도 사용 가능한가요?
A: 현재는 학원(사교육) 특화입니다. 학원법, 근로기준법, 최저임금법 등 학원에 특화된 법적 검증을 제공하기 때문에, 이 전문성이 학온의 핵심 경쟁력입니다.

### Q: 프랜차이즈 학원도 지원하나요?
A: 네. 세션 생성 시 사업자번호(biz_number)로 학원을 식별하기 때문에 프랜차이즈 지점별로 완벽히 분리됩니다. 각 지점은 자기 학원의 계약서만 접근 가능합니다.

### Q: 학원 수가 많아지면 시스템이 감당되나요?
A: 현재 인프라(Supabase + Vercel)는 서버리스 아키텍처로, 트래픽에 따라 자동 확장됩니다. 수천 학원까지는 플랜 업그레이드만으로 대응 가능하며, 그 이상 규모에서는 단계적으로 인프라를 확장할 계획입니다.
