# 학온(HAGON) 데이터맵

화면별 데이터 소스를 정리한 문서입니다.

---

## 데이터 소스 유형

| 유형 | 설명 |
|------|------|
| 🔵 Supabase DB | PostgreSQL 데이터베이스 |
| 🟣 Zustand (Local) | 브라우저 localStorage 기반 상태관리 |
| 🟢 정적 상수 | 코드 내 인라인 상수 |
| 🟠 미구현 (Mock) | 아직 구현되지 않은 데이터 |

---

## 화면별 데이터맵

| 화면 | 데이터 소스 | 상세 |
|------|-----------|------|
| 랜딩 페이지 | 🟢 정적 | 리스크 진단 질문 데이터 (인라인 상수) |
| 로그인 | 🔵 Supabase Auth | supabase.auth.signUp / signInWithPassword |
| 대시보드 | 🔵 Supabase DB | contracts 테이블 (user_id로 필터) |
| 마법사 전체 | 🟣 Zustand (persist) | wizard-store.ts — Academy, Instructor, ContractPeriod, WorkCondition, Salary, Protection |
| 마법사 complete | 🔵 Supabase DB | contracts 테이블 INSERT (contract_data JSONB) |
| 전자서명 | 🔵 Supabase DB | contracts 테이블 UPDATE (signed_at) + react-signature-canvas |
| 계약서 조회 | 🔵 Supabase DB | contracts 테이블 SELECT by id |
| 파트너 런칭 | 🔵 Supabase DB | partner_sessions 테이블 + supabase.auth.verifyOtp |
| PDF 생성 | 🔵 API Route | html2canvas → jspdf (서버사이드) |

### Phase 3 — 인사서류 관리

> 출처: `학온_인사서류_카테고리_v1.docx` — 5개 서랍, 26종+ 서류

| 화면 | 데이터 소스 | 상세 |
|------|-----------|------|
| 인사서류 메인 (/documents) | 🔵 Supabase DB | document_categories 테이블 (5개 서랍), hr_documents 테이블 (academy_id로 필터) |
| 서류 작성 (/documents/create/[type]) | 🟣 Zustand (Local) + 🔵 Supabase DB | document_templates에서 필드 정의 조회, Zustand persist로 임시저장, 완료 시 hr_documents INSERT |
| 서류 조회 (/documents/view/[id]) | 🔵 Supabase DB | hr_documents 테이블 SELECT by id |
| 서류 PDF 출력 | 🔵 API Route | html2canvas → jspdf (계약서와 동일 패턴) |
| 서류 서명 (/documents/sign/[id]) | 🔵 Supabase DB | hr_documents UPDATE (signed_at) + react-signature-canvas |
