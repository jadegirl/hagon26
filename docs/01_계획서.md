# 학온(HAGON) 전자계약 플랫폼 고도화 개발계획서

학원 강사 근로계약서의 법적 리스크 자동 검증 및 전자 서명 통합 시스템 구축을 위한 단계별 로드맵

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 학온(HAGON) |
| 유형 | 학원 전자계약 플랫폼 |
| 목표 | 법적 리스크 자동 검증 + 전자 서명 + 계약 관리 통합 |
| 타겟 사용자 | 학원 원장, 상위 법인 관리자 |
| 비즈니스 모델 | 월 구독료 (체험판 → 프로 → 엔터프라이즈) |
| 기술 스택 | Next.js 14, TypeScript, Tailwind CSS, Supabase, Zustand |
| **핵심 차별점** | **1,200건 분쟁 데이터 기반 18가지 법적 자동 검증** |

---

## 2. 고도화 범위 (Phase별)

### Phase 1: MVP 고도화
- 계약서 UI/UX 개선
- 대시보드 고도화
- 기존 기능 안정화

### Phase 2: 관리 시스템
- 총관리자/학원관리자 시스템
- 회원체계 (법인 → 하위 학원)
- 구독 관리

### Phase 3: 확장
- 계약서 유형 추가 (시급제 등)
- 인사서류 확장
- 파트너 API 고도화

---

## 3. 회원 체계

| 순서 | 역할 | 설명 |
|------|------|------|
| 1 | 슈퍼관리자 | 학온 운영팀 (전체 시스템 제어) |
| 2 | 법인 관리자 | 학원 법인 대표 (소속 학원 통합 관리) |
| 3 | 학원 관리자 | 개별 학원 원장 (계약 생성 및 관리) |
| 4 | 강사 | 서명/열람 전용 사용자 |

---

## 4. 핵심 기능

| 기능 | 설명 |
|------|------|
| 계약서 마법사 | 8단계 스텝 가이드 |
| 전자서명 | 카카오톡/링크 발송 |
| D-Day 알림 | 계약 만료일 관리 |
| 법적 리스크 검증 | 18가지 항목 자동 체크 |
| 파트너 API | SSO + Webhook 연동 |
| PDF 다운로드 | 표준 양식 출력 |

## 5. 기술 아키텍처

### Frontend
- Next.js 14 App Router

### Backend & Infra
- Supabase (PostgreSQL)
- RLS & Auth

### Specialized Tech
- html2canvas + jspdf
- react-signature-canvas
- Kakao SDK Integration
