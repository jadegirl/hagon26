# 학온(HAGON) — 개발계획서

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 학온(HAGON) |
| **유형** | 학원 전자계약 · 인사서류 플랫폼 |
| **목표** | 학원 강사 근로계약서의 법적 리스크 자동 검증 + 전자 서명 + 인사·노무 서류 통합 관리 시스템 구축 |
| **타겟 사용자** | 학원 원장, 상위 법인 관리자 |
| **비즈니스 모델** | 월 구독료 (체험판 → 프로 → 엔터프라이즈) |
| **핵심 차별점** | 1,200건 분쟁 데이터 기반 18가지 법적 자동 검증 |

### 1.1 프로젝트 비전

"**학원 강사 계약, 법적 분쟁 전에 AI가 먼저 검증하는 시대**"

- 학원 강사 근로계약서 작성 시 발생하는 법적 리스크를 사전에 자동 검증
- 1,200건 실제 분쟁 데이터를 학습한 18가지 검증 항목으로 법적 안전성 확보
- 전자서명 → 계약 관리 → 만료 알림까지 계약 전 생명주기를 하나의 플랫폼에서 처리
- 법인 → 학원 계층 구조로 다지점 학원의 통합 관리 지원

## 2. 사이트맵

```

│   │   ├── 계약서 목록 (진행중/완료 탭)

│   │   ├── Step 4: 계약기간 설정

│   │
│   ├── 전자서명 (/sign/[id])
│   └── 계약서 조회 (/view/[id]) — 읽기 전용
│
│   ├── 📄 인사서류 관리 (/documents) — Phase 3
│   │   ├── 서류함 목록 (5개 카테고리)
│   │   ├── 서류 작성 (/documents/create/[type])

├── 🔗 파트너 연동

│       ├── /api/partner/session — 세션 토큰 발급
│       ├── /api/partner/verify — 토큰 검증
│       ├── /api/partner/sso — 단일 로그인
│       ├── /api/partner/callback — 계약 완료 콜백
│       └── /api/partner/cancel — 취소 처리
│
├── 🛡️ 총관리자 (Phase 2)
│   ├── 대시보드 (/admin)
│   ├── 학원 관리 (/admin/academies)
│   ├── 구독/요금 관리 (/admin/subscriptions)
│   └── 사용자 관리 (/admin/users)
│
├── 🏫 학원 관리자 (Phase 2)
│   ├── 학원 대시보드 (/academy/[id]/dashboard)
│   ├── 계약서 관리 (/academy/[id]/contracts)

├── 📖 개발문서 (/devdocs)
│
└── 📄 기타
    ├── /api/pdf — PDF 생성
    └── /api/signature — 서명 데이터 처리
```

### 2.2 URL 설계

| 페이지 유형 | URL 패턴 | 설명 |
|-------------|----------|------|
| 랜딩 | `/` | 서비스 소개, 기능, 가격, 리스크 진단 |
| 인증 | `/login` | 로그인/회원가입 |
| 대시보드 | `/dashboard` | 사용자 계약서 목록 |
| 마법사 | `/wizard/type-a/step-{N}` | 계약서 작성 단계별 |
| 전자서명 | `/sign/[id]` | 계약 당사자 서명 |
| 계약서 조회 | `/view/[id]` | 읽기 전용 계약서 |
| 총관리자 | `/admin/*` | 시스템 전체 관리 |
| 학원관리자 | `/academy/[id]/*` | 개별 학원 관리 |
| 인사서류 | `/documents/*` | 인사·노무 서류 관리 (Phase 3) |
| 서류 작성 | `/documents/create/[type]` | 서류 유형별 작성 마법사 |
| 서류 조회 | `/documents/view/[id]` | 작성된 서류 열람 |

## 4. 회원 체계

### 4.1 역할 구조

```
슈퍼관리자 (학온 운영팀)
  └── 법인 관리자 (학원 법인 대표)
       └── 학원 관리자 (개별 학원 원장)
            └── 강사 (서명/열람 전용)
```

### 4.2 역할별 권한

| 역할 | 계약서 작성 | 인사서류 작성 | 계약서 조회 | 서명 | 학원 관리 | 구독 관리 | 시스템 관리 |
|------|:---------:|:----------:|:---------:|:---:|:--------:|:--------:|:----------:|
| 슈퍼관리자 | ✅ | ✅ 전체 | ✅ 전체 | ❌ | ✅ 전체 | ✅ | ✅ |
| 법인 관리자 | ✅ | ✅ 소속 법인 | ✅ 소속 법인 | ❌ | ✅ 소속 학원 | ✅ | ❌ |
| 학원 관리자 | ✅ | ✅ 소속 학원 | ✅ 소속 학원 | ❌ | ❌ | ❌ | ❌ |
| 강사 | ❌ | ❌ (사직서 제출만) | ✅ 본인 | ✅ | ❌ | ❌ | ❌ |

## 8. 데이터 정책

### 8.1 계약서 데이터 관리

- 모든 계약서 데이터는 Supabase PostgreSQL에 JSONB 형태로 저장
- RLS(Row Level Security)로 사용자 본인의 계약서만 접근 가능
- 계약서 삭제 시 소프트 삭제 (archived) — 법적 보관 의무 고려

### 8.4 인사서류 데이터 관리

- 인사서류는 계약서와 동일하게 JSONB 형태로 저장, RLS로 접근 제어
- 서류 카테고리(5개 서랍)와 템플릿(26종+)은 별도 테이블로 관리
- 각 서류의 입력 필드는 DB 스키마 설계의 기준이 됨 (향후 변경 가능)
- 법률 문구는 별도 검수 단계에서 추가 (현재는 카테고리 및 입력 필드 확정이 최우선)
### 8.2 개인정보 보호

- 강사 개인정보 (이름, 연락처, 주민번호 앞자리) 암호화 저장
- 전자서명 데이터 별도 보관
- 파트너 API 연동 시 최소 정보만 전달

### 8.3 데이터 백업

- Supabase 자동 백업 (Point-in-Time Recovery)
- 중요 계약서 PDF 별도 스토리지 보관

## 9. 파트너 연동 (API)

### 9.1 연동 방식

| 방식 | 설명 | 현재 상태 |
|------|------|----------|
| **SSO** | 파트너사 앱 → 학온 자동 로그인 (토큰 기반) | ✅ 구현 완료 |
| **Webhook** | 계약 완료 시 파트너사에 콜백 알림 | ✅ 구현 완료 |
| **세션 토큰** | 10분 만료 일회성 토큰으로 보안 확보 | ✅ 구현 완료 |

### 9.2 연동 대상

| 파트너 | 유형 | 연동 내용 |
|--------|------|----------|
| **TeamPro** | 학원관리 프로그램 | SSO + 계약 완료 콜백 + 취소 처리 |
| 추가 파트너 | 미정 | Phase 3에서 API 고도화 후 확대 |
