# 학온(HAGON) 데이터베이스 설계

Supabase PostgreSQL 테이블 구조 및 관계 정의

---

## 현재 테이블 (MVP)

### contracts
계약서 핵심 데이터 저장

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| user_id | UUID (FK) | auth.users 참조 |
| contract_data | JSONB | 마법사 입력 전체 데이터 |
| status | TEXT | draft, completed, signed |
| created_at | TIMESTAMPTZ | 생성일시 |
| updated_at | TIMESTAMPTZ | 수정일시 |

> **Notes:** RLS: 자기 계약서만 CRUD 가능. 인덱스: user_id, status, created_at DESC

### partners
파트너사 관리 (플랜, 쿼터, 인증키)

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| name | TEXT | 파트너사 명칭 |
| api_key | TEXT | 인증용 키 |

### partner_sessions
세션 토큰 (10분 만료, 일회성)

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| partner_id | UUID (FK) | partners 참조 |
| token | TEXT | 일회성 토큰 |
| expires_at | TIMESTAMPTZ | 만료 시간 |

### partner_contracts
파트너 경유 계약 추적

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| contract_id | UUID (FK) | contracts 참조 |
| partner_id | UUID (FK) | partners 참조 |

## 고도화 예정 테이블 (Phase 3) — 인사서류

> 출처: `학온_인사서류_카테고리_v1.docx` — 5개 서랍, 26종+ 서류
> 각 서류의 입력 필드는 DB 스키마 설계의 기준이 됨 (향후 변경 가능)

### document_categories
서류 카테고리 (5개 서랍) 마스터 테이블

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| code | TEXT (UNIQUE) | 서랍 코드 (hiring, attendance, resignation, certification, legal) |
| name | TEXT | 카테고리명 (채용 및 계약 단계, 근태 및 복무 관리 등) |
| display_order | INT | 정렬 순서 (1~5) |
| description | TEXT | 카테고리 설명 |
| max_documents | INT | 최대 서류 수 (여유 공간 포함, 기본 ~12) |
| created_at | TIMESTAMPTZ | 생성일시 |

> 초기 데이터: 01 채용 및 계약(8종), 02 근태 및 복무(6종), 03 퇴직 및 계약 종료(6종), 04 기타 증명 및 서약(5종), 05 법무 컨설팅 요청(1종)

### document_templates
서류 템플릿 (26종+) — 입력 필드 스키마 정의

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| category_id | UUID (FK) | document_categories 참조 |
| code | TEXT (UNIQUE) | 서류 코드 (instructor_contract, freelancer_contract, resignation_letter 등) |
| name | TEXT | 서류명 (강사근로계약서, 프리랜서강사계약서 등) |
| field_schema | JSONB | 입력 필드 정의 (name, type, required, options 등) |
| requires_signature | BOOLEAN | 서명 필요 여부 |
| legal_note | TEXT | 법적 유의사항 (예: 학원법 제22조) |
| display_order | INT | 카테고리 내 정렬 순서 |
| is_active | BOOLEAN | 활성화 여부 (default: true) |
| created_at | TIMESTAMPTZ | 생성일시 |
| updated_at | TIMESTAMPTZ | 수정일시 |

> 예시 field_schema:
> ```json
> [
>   { "name": "instructor_name", "label": "강사 성명", "type": "text", "required": true },
>   { "name": "teaching_hours", "label": "강의 시간", "type": "number", "required": true },
>   { "name": "payment_type", "label": "급여 유형", "type": "select", "options": ["고정급", "비율제", "시간제"], "required": true }
> ]
> ```

### hr_documents
실제 작성된 인사서류 인스턴스

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | 고유 식별자 |
| template_id | UUID (FK) | document_templates 참조 |
| academy_id | UUID (FK) | academies 참조 |
| created_by | UUID (FK) | auth.users 참조 (작성자) |
| target_user_id | UUID (FK) | auth.users 참조 (대상자 — 강사 등, nullable) |
| document_data | JSONB | 서류 입력 데이터 (field_schema 기준) |
| status | TEXT | draft, completed, signed, archived |
| signed_at | TIMESTAMPTZ | 서명 일시 (nullable) |
| created_at | TIMESTAMPTZ | 생성일시 |
| updated_at | TIMESTAMPTZ | 수정일시 |

> **Notes:**
> - RLS: 소속 학원(academy_id) 단위로 접근 제어
> - 인덱스: academy_id, template_id, status, created_at DESC
> - 소프트 삭제 (archived) — 법적 보관 의무 고려
> - document_data는 계약서(contracts)의 contract_data와 동일 패턴 (JSONB)

### 테이블 관계도 (Phase 3 추가분)

```
document_categories (5행)
  └── document_templates (26행+)    [category_id → document_categories.id]
       └── hr_documents (N행)        [template_id → document_templates.id]
            ├── academies               [academy_id → academies.id]
            ├── auth.users              [created_by → auth.users.id]
            └── auth.users              [target_user_id → auth.users.id]
```
